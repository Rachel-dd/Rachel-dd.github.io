GDW Exercise 5.6: Hidden Information

Exercise: Many strategy games have open information structures that allow the players access to perfect information about the game state.
Examples are chess, checkers, Go, mancala, etc. Take a game with an open information structure and change the system so that there is an element of hidden information.
You might need to add new concepts to the game to accomplish this. Test your new design. How does adding hidden information change the nature of the strategy?
Why do you think this is so?

CHECKERS
change: make the most inside ball invisibal for the components. So this ball can be overlaped by components and the owner(just treat it as air, can’t jump across but can overlap).
But this invisibal ball can jump across other visibal balls.
In order to do this, we need a fair system who can see all the players’ invisibal ball and make sure the player who wins have successfully put it in right place.

What may this hidden information affect the nature of the strategy?
The time when a player moves this ball is known by others, but don’t know the position. So when we found someone is moving the invisibal ball, there might be a long way for it.
Otherwise, the player won’t take the risk of being known to move just a little steps. So when one players is moving the invisibal ball, others may guess from the possible long way
where this ball is, and try to break the way(maybe). Otherwise, if someone is moving the invisibal, this might indicate that the other balls of the player can’t move so long(<the invisibal)
this makes it possible to guess the position of the invisible as well.
